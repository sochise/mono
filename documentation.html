01. Architektura Aplikace

MONO je moderní Single Page Application (SPA) postavená na čistých webových standardech bez závislosti na build procesech nebo frameworcích.

Stack

  • Jádro: Vanilla JavaScript (ES6+)
  • UI: HTML5 + Tailwind CSS (via CDN)
  • Audio: Web Audio API (Generativní hnědý šum)
  • Video: YouTube IFrame API
  • Data: LocalStorage / SessionStorage

Vlastnosti

  • Zero-Backend: Aplikace nepotřebuje serverovou část.
  • Offline-First: Kromě YouTube API funguje kompletně offline.
  • Reaktivní State: Vlastní implementace reaktivního stavu.

Volba "No-Framework" přístupu (bez React/Vue) zajišťuje maximální výkon, nulovou konfigurační zátěž a dlouhodobou udržitelnost kódu.

03. Aplikační Logika

State Management

Stav aplikace je držen v centrálním reaktivním objektu state. Změny stavu probíhají výhradně přes funkci setState(), která spouští vedlejší efekty (překreslení UI, ukládání dat).

const state = {
    phase: 'PREP', // 'PREP' | 'WORK' | 'BREAK'
    timeLeft: 21.25 * 60,
    thoughts: [],
    feedback: [],
    isEvening: false,
    // ... další stavové proměnné
};

Časovač (The Heartbeat)

Časovač běží v hlavním vlákně pomocí setInterval(1000ms). Pro zajištění přesnosti i při neaktivním tabu (throttling prohlížeče) se porovnává Date.now() s očekávaným časem konce (drift correction).

Audio Engine

Generování "Hnědého šumu" (Brown Noise) využívá Web Audio API. Šum je syntetizován v reálném čase matematickou funkcí, což eliminuje potřebu stahovat velké zvukové soubory a umožňuje plynulé přechody (fade-in/out).

05. UI/UX Principy

Design sleduje principy Neuro-Ergonomie:

  • Redukce Kognitivní Zátěže: Rozhraní skrývá vše, co není relevantní pro aktuální fázi (koncept "Progressive Disclosure"). V režimu FLOW zmizí navigační prvky.
  • Cirkadiánní Adaptivita: Přes den aplikace používá chladné modré tóny (podpora bdělosti). Po 18:00 automaticky přechází do "Jantarového módu" (filtrace modrého světla, podpora melatoninu).
  • Barevné Kódování Stavů:
    • Modrá = Fokus / Práce
    • Jantarová = Odpočinek / Regenerace
    • Šedá = Neaktivita / Příprava

06. Bezpečnost a Data

Aplikace dodržuje striktní přístup Local-First & Privacy-By-Design.

Ukládání Dat

Veškerá uživatelská data (úkoly, myšlenky, historie) jsou ukládána pouze v prohlížeči uživatele pomocí API localStorage. Žádná data neodchází na vzdálený server.

Autorizace

Vzhledem k lokální povaze aplikace neexistuje systém uživatelských účtů ani přihlašování. Bezpečnost dat je vázána na fyzický přístup k zařízení.

Externí Spojení

Jediná externí komunikace probíhá vůči službě YouTube (Google) pro načítání videí. Uživatelé jsou vázáni podmínkami služby YouTube.
